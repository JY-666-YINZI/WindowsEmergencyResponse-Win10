# 文件分析
## 需要查清的问题
+ 攻击者的 IP 地址（通常两个）
+ 攻击者的 webshell 文件名
+ 攻击者的 webshell 连接密码
+ 攻击者的 QQ 号
+ 恶意服务器地址（C2 / FRP 等）
+ 攻击者是如何入侵的（选择题常见）
+ 攻击者创建的隐藏用户名

## 二、寻找入侵路径与开放的服务
### 1. 日志分析与爆破 IP 定位
查看 Web 中间件日志。  
若不知道使用了哪种中间件，可先从“小皮”（phpStudy）等集成环境入手，查找 `access.log`。  
重点关注 **access.log**，error.log 可暂时忽略。

+ 思路：日志中出现大量高频、非人工操作的请求（如目录扫描、密码爆破），即可判定相关 IP 存在攻击行为。
+ **第一个攻击者 IP**：`access.log` 中一直在爆破的 IP，因为不像正常人工访问频率，大概率是扫描/爆破，只要有攻击行为就是攻击者。

### 2. 远程登录 IP 查找
+ **第二个攻击者 IP**：可能是通过远程登录（RDP、第三方远控如向日葵/ToDesk、隧道代理）成功登录的 IP。
+ 打开 `eventvwr.msc`（事件查看器），在 **Windows 日志 → 安全** 中筛选登录事件（如 4624），查找可疑 IP（技巧：直接在日志中搜索 “192.168” 或公网 IP 段快速定位）。

### 3. 使用分析工具
+ 可利用蓝队应急工具箱中的 **Windows 日志分析工具** 自动生成分析报告。
+ 如工具效果不佳，再转为手工分析，核心还是 **access.log** 和行为关联。

### 4. 寻找 Webshell
+ 使用 **D盾** 等扫描器扫描网站目录，定位攻击者留下的木马文件，得到 **webshell 文件名**。
+ 找到后分析文件内容，确认连接密码（如蚁剑、冰蝎流量特征）。
+ 不放心可上传到“微步在线”等平台跑沙箱，进一步确认是否木马。

### 5. 查找攻击者 QQ 号
+ 进入 `C:\Users\<用户名>\Documents\Tencent Files`（或安装目录），查看目录下的文件夹名，QQ 文件夹通常以 QQ 号命名。

### 6. 查找恶意服务器地址
+ 重点检查隧道搭建工具的配置文件（如 frp、nps、ngrok 等），配置文件中会有 **服务器 IP 和端口**。
+ **如何定位配置文件：**
    - 若攻击正在进行，直接使用 `tasklist` 查看进程，定位恶意进程路径。
    - 若为事后排查，可借助 Everything 按时间范围搜索，快速找到近期新建/修改的可疑配置。

### 7. 确定入侵投放方式
已知小皮环境开放了 FTP 服务，而 Web 日志中没有文件上传（upload）记录，也没有直接访问 webshell 的日志，则需考虑其他投递方式：

+ 数据库外联：检查 MySQL 的 `my.ini` 是否允许外部连接。
+ FTP 上传：直接查看 FTP 日志，看是否有上传行为。

### 8. 分析数据库与 FTP 日志
+ 检查 `my.ini` 数据库配置，若没有异常外联配置，则重点转向 FTP 日志。
+ 发现 FTP 日志中大量状态码显示密码爆破，最终爆破成功并上传了一个 `system` 文件（或其他 webshell 文件）。

### 9. 查找隐藏用户
**方法一：事件查看器**  
打开 `eventvwr.msc`，进入 **Windows 日志 → 安全**，根据状态码过滤：

+ 4720 – 创建用户
+ 4732 – 添加到本地安全组
+ 4728 – 添加到全局安全组

**方法二：注册表**  
打开 `regedit`，检查隐藏用户的注册表项（特殊路径如 `HKEY_LOCAL_MACHINE\SAM\SAM\Domains\Account\Users\Names`，需提权查看）。

## 三、深入分析隐藏用户与远程登录
### 1. 通过事件查看器核查隐藏用户
+ 使用 `eventvwr.msc` 查找对应 **TargetSID**，确认用户何时被创建、何时加入管理组。
+ **小技巧**：通过日期缩小搜索范围，搜索 4720 创建用户事件；或搜索 “192.168.” 等内网片段快速定位异常操作。

### 2. 确认远程登录 IP 和端口
+ 基于已确定的时间窗口，筛选成功登录日志（4624），并结合登录类型（如远程桌面 10、网络登录 3）确定攻击者使用的远程 IP。
+ 重点关注 **administrator** 或其他高权限账户的成功登录事件。
